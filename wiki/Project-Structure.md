# Project Structure

Understanding the organization of this research template.

## Directory Layout

```
AI_workshop/
├── data/                    # Data files
│   ├── raw/                # Original, immutable data
│   │   ├── sample_data.csv
│   │   └── generate_data.py
│   └── processed/          # Cleaned/transformed data (gitignored)
│
├── scripts/                # Analysis code
│   ├── analysis.py         # Main analysis script
│   ├── utils.py           # Helper functions
│   └── requirements.txt   # Python dependencies
│
├── output/                 # Generated outputs (gitignored)
│   ├── figures/           # PDF and PNG plots
│   └── tables/            # LaTeX formatted tables
│
├── tex/                    # LaTeX documents
│   ├── paper/             # Research paper
│   │   ├── paper.tex
│   │   └── references.bib
│   └── slides/            # Beamer presentation
│       └── workshop_slides.tex
│
├── .github/               # GitHub specific files
│   └── copilot-instructions.md  # AI agent guidance
│
├── Makefile               # Automation workflow
├── README.md             # Project documentation
├── AGENTS.md             # Instructions for AI coding agents
└── .gitignore            # Files to exclude from Git
```

## Key Directories

### `data/`
**Purpose:** Store all project data.

- **`raw/`**: Original data files that should **never be modified**
  - Commit to Git if files are reasonable size (< 10MB)
  - Include data generation scripts if using synthetic data
  
- **`processed/`**: Transformed/cleaned datasets
  - Generated by scripts, not manually
  - Add to `.gitignore` (should be reproducible)

### `scripts/`
**Purpose:** All analysis and data processing code.

Key files:
- `analysis.py` - Main analysis pipeline
- `utils.py` - Reusable helper functions
- `requirements.txt` - Python package dependencies

**Best practices:**
- Use type hints for all functions
- Include docstrings (NumPy format)
- Follow PEP 8 style guide
- Modular design (separate concerns)

### `output/`
**Purpose:** All generated figures and tables.

- **`figures/`**: Save both PDF (for papers) and PNG (for preview)
- **`tables/`**: LaTeX formatted tables (.tex files)

**Important:** This entire directory is gitignored - outputs should be regenerated from scripts.

### `tex/`
**Purpose:** LaTeX source documents.

- **`paper/`**: Research paper manuscript
  - Include bibliography (.bib file)
  - Use `\input{}` for tables
  - Use `\includegraphics{}` for figures
  
- **`slides/`**: Beamer presentation slides

### `.github/`
**Purpose:** GitHub and AI agent configuration.

- `copilot-instructions.md` - Guides AI assistants about your project
  - Coding standards
  - Project conventions
  - Workflow patterns

## File Naming Conventions

### Data Files
- Use snake_case: `sample_data.csv`
- Include date prefixes for versions: `2024_11_experiment_results.csv`
- Be descriptive: `participant_demographics.csv` not `data1.csv`

### Scripts
- Snake_case for Python: `analysis_script.py`
- Descriptive names: `regression_models.py` not `script2.py`

### Outputs
- Figures: `regression_plot.pdf`, `distribution_histogram.png`
- Tables: `summary_statistics.tex`, `regression_results.tex`

## Special Files

### `Makefile`
Orchestrates the entire workflow. See [Makefile Automation](Makefile-Automation) for details.

### `.gitignore`
Specifies files Git should ignore:
- Generated outputs (`output/`)
- Python cache (`__pycache__/`)
- LaTeX auxiliary files (`*.aux`, `*.log`)
- Large data files
- API keys and credentials

### `README.md`
Project overview and quick start guide for newcomers.

### `AGENTS.md`
Detailed instructions for AI coding agents working in the repository.

## Design Principles

1. **Reproducibility**: Everything generated from scripts
2. **Version Control**: Source code and raw data tracked, outputs regenerated
3. **Automation**: One command (`make all`) rebuilds everything
4. **Modularity**: Separate data, code, and documents
5. **Documentation**: Clear structure, descriptive names, inline comments
